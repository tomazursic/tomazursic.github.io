Python Idioms in Rust

The goal is to catch matches with Python, from some point of view this notes are continuation from reading Python Idioms in Rust

Variables

Example: Python immutable

let mut a = 1;
a = 2;
let b = a;
println!("a: {}", a);
println!("b: {}", b);

..
a: 2
b: 2

Enumerate

A common pattern in when iterating over lists in Python is to use enumerate to have get both the index and value of list elements:

>>> arr = [1, 2, 3, 4]
>>> for x, i in enumerate(arr):
...     print(f"{x}, {i}")
...
0 1
1 2
2 3
3 4
>>>

Rust iterators also support enumerate! (And you don’t even have to learn a new name for the same functionality)

// using enumerate for iterating with index
let coords = vec!["x", "y", "z"];
for (x, i) in coords.iter().enumerate() {
    println!("{}, {}", x, i);
}

...
0, x
1, y
2, z

Zip

>>> arr = [1, 2, 3, 4]
>>> arr2 = ["a", "b"]
>>> for a, b in zip(arr, arr2):
...     print(f"{a}, {b}")
...
1, a
2, b

let positions = vec![3.44, -91.02];
for z in coords.iter().zip(positions) {
    println!("{:?}", z)
}

...
("x", 3.44)
("y", -91.02)

Tuples

Rust tuples are sized at compile-time, unlike Python tuples which are dynamic. In Rust “destructuring” in python is called “unpacking”.

Tuples are mutable

Example: Python tuple

let mut tup = (1, 2, 3)
tup.0 = 6

Closures

Anonymous functions

Example: Python lambda-functions

let mut arr: Vec<i32> = vec![10, 15, 3];
arr.sort_by_key(|a| (10 - a).abs());

Rust closures can also be multiline functions using brackets, which is quite handy:

let mut arr = vec![10, 15, 3];
arr.sort_by_key(|a| {
    let intermediate_value = some_function(a);
    intermediate_value + 10
});

The big difference between closures and other functions is that closures capture their environment. This means that we can refer to variables outside the closure from the closure. Access variable in scope.

let a = 10;
let plus_a = |x| x + a;
plus_a(5);

...
15

First-Class Functions

Functions and closures can be used as argument to other functions Functions are not objects,

fn plus_one(x: i32) -> i32 {
    x + 1
}

fn map_on_vec(vec: &Vec<i32>, func: fn(i32) -> i32) -> Vec<i32>
{
    let mut new_vec = Vec::new();
    for i in 0..vec.len() {
        new_vec.push(func(vec[i]));
    }
    new_vec
}

let vec = vec![1, 2, 3];
println!("{:?}", map_on_vec(&vec, plus_one));
// [2, 3, 4]

// Equivalently:
println!("{:?}", map_on_vec(&vec, |x| x + 1));

Ownership

Enforce single resource owner.

It is assumed that every value is represented by a variable binding (you may say name used in code) which is called its owner and lives in a specific scope. The owner is assigned for the first time when value is created, and the value will be destroyed when the owner goes out of scope.

Ownership can be also moved to another variable binding, then you have a new owner but you cannot use original binding any more.

// This code fails
let s1 = "Hello world!".to_string();
let s2 = s1;
println!("{}", s1);
